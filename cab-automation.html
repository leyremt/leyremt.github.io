Professional Project · Product Owner

CAB Briefing Automation

AI GitHub Copilot Change Advisory Board Plants
Stack: VS Code Copilot agents + skills · Python (openpyxl) · ServiceNow REST · Model Context Protocol (itsm-next MCP) · YubiKey / Azure AD auth

Problem

Every month the Global IT Change Advisory Board must decide which ServiceNow changes are relevant for our department, and record the affected applications, feedback groups and plants across four agenda columns.

The task was manual, repetitive and error-prone, with Excel as the main tool: for dozens of changes, a single coordinator opened each record, downloaded an affected-application spreadsheet, interpreted evolving rules and hand-typed four columns. This created transcription errors, inconsistency and severe key-person risk — the classification rules lived in one person's head. On average it required up to 4 hours of manual work each week.

Solution

I designed and delivered an agentic automation that turns the process into a repeatable, rule-based pipeline with a human only at specific decision points. A single orchestrator agent runs three stages:

1 · Prepare

Fetches change metadata and the correct attachment from ServiceNow, parses heterogeneous spreadsheets (multi-sheet, name and plant normalization), applies the codified classification rules, and writes the agenda columns.

2 · QA

Independently re-derives every value and diffs it, flagging only true edge cases (OK / MISMATCH / REVIEW), including cross-domain dependency checks.

3 · Tag

Tags relevant changes in production ServiceNow after an explicit go/no-go.

Human intervention is reduced to three checkpoints: one authentication, one go/no-go before the production write, and review of only the specific ambiguous rows.

My contribution as Product Owner

  • Documented and collected tribal knowledge into explicit, unit-tested classification rules (the single source of truth), gathered from the most relevant stakeholders.
  • Defined the agent architecture (orchestrator + specialized prepare / QA / tag subagents) and the human-in-the-loop checkpoints.
  • Authored the product backlog — 2 developers, 2 epics, 5 user stories with acceptance criteria, story-point estimates, and a MoSCoW / value-vs-effort prioritization.
  • Embedded security-by-design: secrets kept out of version control, least-privilege access, and a mandatory confirmation before any production change.

Impact

Faster cycle time and near-elimination of manual transcription errors.

Consistency & auditability through rule-driven, independently QA'd output.

Reduced key-person risk — the process is now documented, testable and handover-ready.

A clear roadmap to full autonomy — non-interactive auth, scheduling, confidence-gated auto-tagging and a self-service UI.
